Goal

Build “Zedwriter,” a student research assistance web app for Zambia. Deliver a zippable project with no proprietary builder runtime—pure HTML, CSS, and vanilla JS for the client; Firebase for backend services.

Tech & Architecture

Frontend: HTML, CSS, JS (no frameworks).

Backend: Firebase Auth, Firestore, Cloud Storage, Cloud Functions, Callable + HTTPS APIs, Firestore & Storage Security Rules.

Payments: Abstract payment module + webhook handler (supports mobile money providers via a single /payments/webhook endpoint; provider SDK can be swapped later).

Hosting: Firebase Hosting (or Vercel static).

Environments: .env.example with keys; do not commit real secrets.

Type-specific:

Proposal/Dissertation: Research Title, Supervisor Name, Supervisor Contact

Assignment: Topic/Title, Field of Study, Instructor Name

Validation: required fields; email/phone regex; max 500 chars comments.

Step 2 — Files: Multi-upload (PDF/DOCX/PPT/XLSX), ≤20MB each, ≤6 files, progress UI, remove/replace.

Step 3 — Review: Read-only summary; Back/Submit.

Submit → progress bar (“Saving details”, “Uploading files”, “Completed”) → success toast + submission ID + link to Dashboard.

Data Models (Firestore)

users/{uid}: displayName, email, phone, school, role('user'|'admin'), avatarUrl, referralCode, referredBy, points, payoutMethod, createdAt, updatedAt

submissions/{id}: uid, type('proposal'|'dissertation'|'assignment'), commonFields{...}, typeFields{...}, preferredDate, paymentMethod, paymentArrangement, comments, status('pending'|'in_progress'|'under_review'|'completed'), amount, files[ {path,name,size,contentType} ], createdAt, updatedAt

messages/{threadId} + messages/{threadId}/entries/{msgId}: thread meta per user/admin; entries have from('user'|'admin'), type('text'|'file'|'voice'), text, file{...}, createdAt, readBy

payments/{id}: uid, submissionId?, amount, currency('ZMW'), method('mobile_money'|'bank'), provider('providerX'), providerRef, status('initiated'|'confirmed'|'failed'|'refunded'), createdAt, updatedAt

referrals/{id}: ownerUid, event('signup'|'proposal_paid'|'dissertation_paid'), deltaPoints, sourceUid, createdAt

announcements/{id}: title, body, createdAt, createdBy

announcement_reads/{id}: announcementId, uid, readAt

materials/{id}: program, year, type, title, description, file{path,...}, uploadedBy, status('in_review'|'published'|'rejected'), createdAt, updatedAt

file_requests/{id}: uid, program, requestText, status('open'|'closed'), createdAt

Indexes: composite on submissions.status+createdAt, materials.program+year+type, messages.threadId+createdAt, payments.status+createdAt.

Referral Logic

Base: 1 point = K1; withdraw at ≥200 points.

Events:

New signup via referral link → +2 points to referrer.

Referred user pays for proposal → +25.

Referred user pays for dissertation → +50.

Anti-abuse: each event type counted once per referred user; webhooks must verify provider signature; IP/device fingerprint heuristics to flag abuse (store flags in users.{riskScore}).

Validation & Limits

Emails and phone numbers regex; Zambia numbers begin +260…

File types: .pdf,.doc,.docx,.ppt,.pptx,.xls,.xlsx; sizes: ≤20MB (submissions), ≤10MB (chat/file), ≤1MB (avatars).

Comments ≤500 chars; study area ≤120 chars; keywords ≤12 selected.

Rate-limit submission attempts: ≤3/min per IP (Cloud Function check + Firestore counter).

Security (must implement)

Auth: Firebase Auth (Email/Password + Google).

RBAC: Firestore/Functions server-side checks: only owner can read/write their resources; only admins can see everything.

PII: store minimal PII; never expose student IDs in public collections; all downloads require auth & rule check.

Audit: write admin actions to audit_logs (who, what, when).

Rules: deny writes with unexpected fields; validate file metadata on upload (contentType/size).

Webhooks: verify provider signature; idempotent by providerRef; never trust client-side payment “success.”

Performance & A11y

LCP < 2.5s (3G Fast); JS ≤ 150KB gz total per route; images lazy-loaded; compress/crop uploads client-side.

Tab order logical; focus states visible; ARIA labels on interactive elements; prefers-reduced-motion respected.

Firebase Rules (include and deploy)

Firestore (excerpt)

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isSignedIn() { return request.auth != null; }
    function isAdmin() { return isSignedIn() && request.auth.token.admin == true; }
    function isOwner(uid) { return isSignedIn() && request.auth.uid == uid; }

    match /users/{uid} {
      allow read: if isOwner(uid) || isAdmin();
      allow create: if isOwner(uid);
      allow update: if isOwner(uid) || isAdmin();
      allow list: if false;
    }

    match /submissions/{id} {
      allow create: if isSignedIn();
      allow read, update: if isOwner(resource.data.uid) || isAdmin();
      allow delete: if isAdmin();
    }

    match /messages/{threadId} {
      allow read, write: if isAdmin() || (isSignedIn() && request.resource.data.participants.hasOnly([request.auth.uid]));
    }
    match /messages/{threadId}/entries/{msgId} {
      allow read, create: if isSignedIn();
      allow delete: if isAdmin();
    }

    match /payments/{id} {
      allow create: if isSignedIn();
      allow read: if isOwner(resource.data.uid) || isAdmin();
      allow update, delete: if isAdmin();
    }

    match /announcements/{id} {
      allow read: if true;
      allow create, update, delete: if isAdmin();
    }
    match /announcement_reads/{id} {
      allow read, create: if isSignedIn() && (request.resource.data.uid == request.auth.uid);
    }

    match /materials/{id} {
      allow read: if true;
      allow create: if isSignedIn(); // auto-sets status='in_review'
      allow update, delete: if isAdmin();
    }

    match /file_requests/{id} {
      allow read, create: if isSignedIn();
      allow update, delete: if isAdmin();
    }
  }
}


Storage (excerpt)

rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {

    function isSignedIn() { return request.auth != null; }
    function isAdmin() { return isSignedIn() && request.auth.token.admin == true; }

    match /uploads/{uid}/{allPaths=**} {
      allow read: if isAdmin() || (isSignedIn() && request.auth.uid == uid);
      allow write: if isSignedIn()
        && request.resource.size < 20 * 1024 * 1024
        && request.resource.contentType.matches('application/(pdf|msword|vnd.openxmlformats.*)|image/(png|jpeg)|audio/webm');
    }

    match /programs/{program}/{file} {
      allow read: if true;
      allow write: if isAdmin();
    }
  }
}
